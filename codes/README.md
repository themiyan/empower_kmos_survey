# `codes/` — EMPOWER spec-z analysis

Notebooks that audit spectroscopic-redshift coverage for EMPOWER target selection.
All paths are relative to the repository root; run from a fresh clone after setting
up the environment (see the top-level [README](../README.md)).

## Files

| File | What it does |
|------|--------------|
| [`spec_z_analysis.ipynb`](spec_z_analysis.ipynb) | **Main notebook.** Loads the COSMOS (Khostovan+25) and GOODS-S (ASTRODEEP-GS43) baselines, layers in JADES DR4, DJA, DEVILS, VANDELS, MUSE-Wide and MUSE-UDF, builds a unified GOODS-S spec-z table via priority-ordered de-duplication, and produces the per-tier inventory and figures. |
| [`test_spec_cats.ipynb`](test_spec_cats.ipynb) | Exploration / validation: cross-match diagnostics, redshift-agreement and photo-z checks, per-survey breakdowns. Sandbox behind the main notebook. |
| `_apply_dr4_edits.py` | Idempotent helper that rewrites specific cells of `spec_z_analysis.ipynb` (used when refreshing the JADES DR4 layer and per-survey match radii). Locates the notebook via `__file__`, so run it from anywhere. |
| `figures/` | PNG outputs, regenerated by the notebooks. `figures/specz_analysis/` holds the curated `fig0N_*` summary set; the top-level `NN_*.png` are diagnostic plots. |

## Run order

1. Open `spec_z_analysis.ipynb` and run top-to-bottom. It writes figures into
   `figures/specz_analysis/`.
2. `test_spec_cats.ipynb` is optional and standalone — use it for deeper diagnostics.

## Notes for contributors

- Notebook **outputs are stripped** before committing (keeps diffs readable and the
  repo small). Clear outputs again before committing changes, e.g.
  `jupyter nbconvert --clear-output --inplace spec_z_analysis.ipynb`
  (or install `nbstripout` as a git filter).
- Catalogue access goes through the relative `SPEC_DIR = <repo>/catalogues` helper
  defined near the top of the main notebook — keep new loaders consistent with it.
