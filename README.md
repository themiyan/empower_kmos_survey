# EMPOWER KMOS Survey — spec-z inventory & sample selection

Code and supporting catalogues for the **sample-selection** work of the EMPOWER
VLT/KMOS Large Programme. The repository's centre of gravity is the
spectroscopic-redshift (spec-z) audit in [`codes/`](codes/): how much reliable
spec-z already exists for EMPOWER's targets, where the gaps are, and which
external surveys fill them — across the **COSMOS** and **GOODS-S** fields.

## The survey in one paragraph

EMPOWER is a VLT/KMOS Large Programme (LP 26107) targeting **~900 galaxies** with
**M⋆ ≳ 10¹⁰ M⊙** in three redshift tiers (**z ≈ 0.75, 1.6, 2.3**) to study the
interplay between galaxies, their environment, and their gas. Targets are drawn
from COSMOS, GOODS-S, and four clusters. Robust spectroscopic redshifts are
needed both to place galaxies in the correct KMOS band and to map the
large-scale structure they live in — hence this inventory.

## What this repository is

- An **audit** of spec-z availability and gaps per field and per tier
  ([`codes/spec_z_analysis.ipynb`](codes/spec_z_analysis.ipynb)).
- The **catalogues** that audit consumes, each documented with its source.
- **Not** the full survey hub: proposals, presentations, and meeting material are
  intentionally excluded.

## Getting started

```bash
# 1. environment (conda)
conda env create -f environment.yml
conda activate empower-kmos
# ...or with pip:  pip install -r requirements.txt

# 2. run the main audit
jupyter notebook codes/spec_z_analysis.ipynb
```

The notebooks locate catalogues with **relative paths** (repo-root `/catalogues`),
so they work from a fresh clone with no editing. **Git LFS is not required.**

### A note on the data

All catalogues here are public. A few large reference products are *not* committed
to keep the repo light (e.g. the LEGA-C rest-frame spectra HDF5); re-fetch them
using the instructions in the relevant `catalogues/*/_provenance.txt` /
`README.md`. Several VizieR/TAP tables can be regenerated with `astroquery`.

## Science caveats (read before reusing the numbers)

- **Cross-matches are positional only** (sky-separation nearest-neighbour). They do
  not resolve blends or use redshift agreement to confirm a match — treat
  "NEW vs MATCHED" counts as indicative, not definitive.
- **ASTRODEEP-GS43 photo-z is biased high** by ~+0.46 at z > 1, so it is unreliable
  for tier-2/3 selection on its own; lean on spectroscopy in those tiers.
- Redshift quality flags differ between surveys; the unified GOODS-S compilation
  applies a priority-ordered, per-survey match radius (see the notebook).
